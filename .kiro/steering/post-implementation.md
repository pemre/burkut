# Post-Implementation Checklist

After completing any feature or bugfix spec:

1. **Update README.md** — Reflect new features, changed behavior, or updated project structure.
2. **Update steering docs** (`.kiro/steering/`) — Keep `structure.md`, `tech.md`, and `product.md` in sync with any architectural, tooling, or product changes introduced by the implementation.

These updates should be included as tasks in every spec's `tasks.md`.
